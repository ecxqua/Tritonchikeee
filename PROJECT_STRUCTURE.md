# Структура проекта ReID_newts

Документ для быстрого погружения: где что находится и за что отвечает.

## 1) Коротко о проекте

`ReID_newts` — это Telegram-бот для реидентификации тритонов по фото брюшка.

Основной онлайн-пайплайн:
1. Пользователь отправляет фото в Telegram.
2. YOLO-сегментация выделяет брюшко и делает «развёртку» изображения.
3. ViT-модель строит эмбеддинг запроса.
4. Эмбеддинг сравнивается с базой эмбеддингов датасета.
5. Бот возвращает top-N похожих изображений и текстовый отчёт.

## 3) Папка `bot/` — боевое приложение

### `bot/app.py`
Точка входа бота:
- загружает `.env` (токен),
- создаёт `Bot` и `Dispatcher`,
- регистрирует роутеры,
- запускает polling.

Также задаёт ключевые директории:
- `bot/conservation` — входные/промежуточные изображения,
- `bot/results` — итоговые top-картинки и `res.txt`.

### `bot/handlers/`
Логика обработки сообщений и вызов ML-пайплайна.

- `user_private.py`  
  Обработчики Telegram-сообщений:
  - `/start`,
  - загрузка фото/документа,
  - запуск обработки,
  - отправка top-результатов пользователю.

- `model_utils.py`  
  Оркестратор ML-этапов:
  1) вызывает YOLO-обработку (`deployment_yolo`),
  2) вызывает поиск похожих через ViT (`deployment_vit`).

- `utils/deployment_yolo.py`  
  Инференс сегментации и подготовка изображения:
  - загрузка YOLO-сегментационной модели,
  - построение маски,
  - извлечение центральной линии,
  - геометрическая «развёртка» брюшка,
  - сохранение `image_cropped.jpg`.

- `utils/deployment_vit.py`  
  Инференс ReID по эмбеддингам:
  - загрузка ViT-based модели эмбеддингов,
  - вычисление эмбеддинга запроса,
  - загрузка/кэш эмбеддингов базы (`database_embeddings.pkl`),
  - cosine similarity,
  - выбор top-N,
  - копирование результатов в `bot/results/` + запись `res.txt`.

### `bot/dataset_crop/`
Локальная база изображений для поиска похожих.

Текущая структура внутри `dataset_crop_24/`:
- классы видов (`karelin`, `ribbed`),
- внутри — папки отдельных особей (`1`, `2`, ...),
- внутри — изображения.

### `bot/conservation/`
Рабочая папка для входных файлов пользователя и промежуточных артефактов.

### `bot/results/`
Результаты для отправки в Telegram:
- `top1.jpg ... topN.jpg`,
- `res.txt` с описанием найденных совпадений.

## 4) `deployment_scripts/` — standalone/офлайн-скрипты инференса

Содержит отдельные скрипты (`deployment_yolo.py`, `deployment_vit.py`) для локальной обработки вне Telegram-бота.

Используются для:
- отладки,
- массовой пакетной обработки,
- экспериментов с параметрами без запуска бота.

---

## 5) `train_model_vit/` — обучение ReID-модели

- `train_vit.py`  
  Основной скрипт обучения эмбеддинговой модели:
  - ViT backbone + embedding/projection head,
  - triplet/contrastive-подход,
  - hard negative mining,
  - оценка метрик retrieval (например, Precision@1/5).

- `make_csv.py`  
  Генерация CSV-разметки (пути, класс, особь) из структуры датасета.

- `make_viborka.py`  
  Разбиение данных на train/val/test.

---

## 6) Папки с результатами обучения детекции/сегментации

- `yolo11s_seg_23_12/`

Обычно содержат:

Это тренировочные артефакты, которые используются для выбора лучших весов в инференсе.

## 7) Что за что отвечает в проде (краткая карта)

- Взаимодействие с пользователем: `bot/handlers/user_private.py`
- Запуск общего пайплайна: `bot/handlers/model_utils.py`
- Подготовка изображения (сегментация/развёртка): `bot/handlers/utils/deployment_yolo.py`
- Поиск похожих по эмбеддингам: `bot/handlers/utils/deployment_vit.py`
- Точка старта приложения: `bot/app.py`

---

## 8) Что важно помнить при доработке

1. Качество итогового поиска зависит от **обоих** этапов: YOLO-preprocess + ViT retrieval.
2. Любые изменения путей к весам и датасету лучше централизовать (конфиг/переменные окружения).
3. При обновлении модели нужно согласованно обновлять:
   - веса модели,
   - трансформации,
   - формат/кэш эмбеддингов базы.

---

## 9) Быстрый сценарий выполнения запроса в боте

1. Пользователь отправляет фото.
2. Фото сохраняется в `bot/conservation/image.jpg`.
3. `photo_processing()` запускает сегментацию и создаёт `image_cropped.jpg`.
4. ViT считает эмбеддинг и ищет top-5 в базе.
5. В `bot/results/` формируются `top*.jpg` и `res.txt`.
6. Бот отправляет пользователю медиагруппу и текст результата.
