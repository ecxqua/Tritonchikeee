# Система идентификации индивидуальных особей тритонов
## Независимый пайплайн YOLO-ViT
Чтобы начать анализ фотографии с YOLO и ViT:

1. Подтяните зависимости из `requirements.txt`.
2. Веса лежат в папке yolov11s_seg_finetune_v12.
3. Создайте папки `data/input/` и `data/output/`. В папке `data/input/` должна лежать нужная для обработки фотография.
4. Заполните `config/config.yaml` по примеру `config/config.yaml.example`, указав пути, куда вы сохранили нужное изображение и веса моделей, и поместите конфиг в корне репозитория.
5. Запустите ```python -m pipeline.analyse``` из корня репозитория. Начнётся процесс сегментации и идентификации, идентичный работе тг-бота.
6. В выходных файлах в папке `data/output` появятся артефакты обработки.

ВНИМАНИЕ: перед началом рекомендуется очистить базу данных эмбеддингов: `data\dataset_crop\dataset_crop_24\database_embeddings.pkl`.

ВНИМАНИЕ: папка `train` пока что не работает. Обучение YOLO для сегментации не проверено. Обучение ViT для идентификации тоже не проверено.

## ТГ-бот

ТГ-бот - это оболочка над `pipeline`. Бот загружает файл конфигурации, который затем использует для размещения отправленных пользователем фото, документов и для размещения готовых результатов анализа. Затем он подтягивает единственную асинхронную функцию `photo_processing`, передаёт в неё настройки конфигурации и путь к фото, анализ которого нужно произвести (по умолчанию название файла - 'image.jpg'). 

Для запуска тг-бота:

1. Зарегистрируйте бота в BotsFather и получите токен бота.
2. Создайте файл `.env` в корневой директории. Добавьте в него ваш токен бота: TOKEN=YOUR_TOKEN
3. Выполните шаги 1-4 запуска пайплайна без ручного добавления фото в `data/input/`.
4. Запустите ```python -m bot.app``` из корня репозитория. Отправьте боту фотографию для обработки.
