# Система идентификации индивидуальных особей тритонов
## Независимый пайплайн YOLO-ViT
Чтобы начать анализ фотографии с YOLO и ViT:

1. Подтяните зависимости из `requirements.txt`.
2. Скачайте веса обученных моделей: `python -m bot.download_models`
3. Создайте папки `data/output/`.
4. Заполните `config/config.yaml`.
5. Запустите ```python -m tests.test_identification_service``` из корня репозитория. Начнётся процесс сегментации и идентификации, идентичный работе тг-бота.
6. В выходных файлах в папке `data/output` появятся артефакты обработки (дебаг).

ВНИМАНИЕ: перед началом рекомендуется очистить базы данных по инструкциям ниже.

ВНИМАНИЕ: папка `train` пока что не работает. Обучение YOLO для сегментации не проверено. Обучение ViT для идентификации тоже не проверено.

## Базы данных

Существует две базы: `cards.sqlite3` и `database_embeddings.pkl` (faiss-index). В первой в таблице `individuals` хранятся паспорта индивидуальных особей тритонов, в таблице `photos` карточки к каждой фотографии в базе и `embedding_index`, ссылающийся на индекс эмбеддинга в faiss-index, в таблице `uploads` хранятся незавершённые сохранения в бд, в `projects` - проекты (деление карточек на проекты). `individuals` -> `photos` (1 ко многим, одна особь ко многим фото). В faiss-index хранятся только эмбеддинги для каждого фото.

Схема sqlite-бд описана в cards_database.py. В `pipeline/save_new.py` описаны взаимодействия с бд и FAISS-индексом.

Если вы хотите обнулить все базы, то выполните следующие шаги:
1. Удалите `cards.sqlite3` и `database_embeddings.pkl`.
2. Выполните `bash launch.sh`
